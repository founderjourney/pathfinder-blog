## El Contexto

El cliente es un guia turistico que opera entre Polonia y Colombia. Su mercado es muy especifico: polacos interesados en explorar Colombia de manera autentica, fuera de los circuitos turisticos tradicionales.

Pero habia un componente adicional: parte de las ganancias va destinada a apoyar ninos en comunidades de montana. El sitio necesitaba comunicar tanto la oferta turistica como esta mision social.

**El desafio:** Un sitio que funcione perfectamente en 3 idiomas, se vea profesional, cargue rapido en conexiones lentas, y convierta visitantes en clientes via WhatsApp.

## Requerimientos Unicos

1. **Tres idiomas:** Polaco (primario), Espanol, Ingles
2. **5 destinos destacados** de Colombia
3. **Integracion WhatsApp** para contacto directo
4. **Mision humanitaria** visible pero no invasiva
5. **Mobile-first** - muchos usuarios acceden desde celular
6. **Carga rapida** - turistas buscando desde cualquier parte del mundo

## Desafios Tecnicos

### 1. Internacionalizacion (i18n) Hecha Bien

**El problema:** i18n no es solo traducir textos. Es adaptar fechas, monedas, direccionalidad, y mantener todo sincronizado.

**Complejidades especificas:**
- El polaco tiene casos gramaticales que afectan como se construyen oraciones
- Los nombres de destinos se mantienen en espanol (Cartagena, Medellin)
- Las URLs necesitan reflejar el idioma pero ser SEO-friendly

**La solucion:**

Implemente un sistema de i18n con namespaces:

```javascript
// translations/pl.json
{
  "nav": {
    "destinations": "Destynacje",
    "about": "O nas",
    "mission": "Misja"
  },
  "destinations": {
    "cartagena": {
      "title": "Cartagena",  // Se mantiene
      "description": "Magiczne miasto na wybrzezu Karaibów..."
    }
  }
}
```

Cada idioma tiene su propia ruta (`/pl/destynacje`, `/es/destinos`, `/en/destinations`), y un middleware detecta el idioma preferido del navegador para redirect automatico.

### 2. Selector de Idioma No Intrusivo

**El problema:** El selector de idioma tipico (dropdown con texto largo) ocupa demasiado espacio en mobile y rompe el flujo visual.

**La solucion:** Banderas compactas en el header sticky.

Pero las banderas son problematicas - ?la bandera de UK representa ingles? ?Y los australianos? Opte por un diseno hibrido:

- En desktop: Bandera + codigo de idioma (PL, ES, EN)
- En mobile: Solo banderas (mas compacto)
- Tooltip on hover explicando el idioma

El estado de idioma persiste en localStorage para visitas recurrentes.

### 3. Imagenes que Cargan Rapido

**El problema:** Turismo = muchas fotos. Fotos pesadas = sitio lento. Sitio lento = usuarios que abandonan.

**Metricas objetivo:**
- LCP (Largest Contentful Paint) < 2.5s
- Hero image visible en < 1s en 3G

**La solucion:**

1. **Formatos modernos:** WebP con fallback a JPEG
2. **Responsive images:** Diferentes tamanos segun viewport
3. **Lazy loading:** Imagenes below-the-fold cargan on-scroll
4. **Blur placeholders:** Mientras carga la imagen real, mostrar version tiny blurred

```jsx
<picture>
  <source
    srcSet="/images/cartagena-800.webp 800w,
            /images/cartagena-1200.webp 1200w"
    type="image/webp"
  />
  <img
    src="/images/cartagena-800.jpg"
    loading="lazy"
    style={{ backgroundImage: `url(${blurPlaceholder})` }}
  />
</picture>
```

### 4. Integracion WhatsApp que Convierte

**El problema:** Un boton de WhatsApp generico no es suficiente. El mensaje inicial debe ser contextual y en el idioma correcto.

**La solucion:**

Cada CTA de WhatsApp incluye un mensaje pre-poblado basado en:
- El idioma actual del usuario
- La pagina donde hace click
- El destino que estaba viendo

```javascript
function getWhatsAppUrl(context, language) {
  const messages = {
    pl: `Czesc! Interesuje mnie ${context.destination}...`,
    es: `Hola! Me interesa ${context.destination}...`,
    en: `Hi! I'm interested in ${context.destination}...`
  }
  return `https://wa.me/${PHONE}?text=${encodeURIComponent(messages[language])}`
}
```

Esto reduce friccion y aumenta probabilidad de respuesta.

## La Mision Social

El cliente queria destacar su trabajo con comunidades de montana, pero sin que pareciera "greenwashing".

**El enfoque:**
- Una seccion dedicada, no un popup intrusivo
- Fotos reales de las comunidades (con permiso)
- Transparencia sobre como se usan los fondos
- Vinculacion natural: "cuando viajas con nosotros, parte de tu inversion va a..."

La mision esta presente pero no domina - el producto sigue siendo turismo de calidad.

## Lecciones Aprendidas

### 1. i18n desde el dia 0

Agregar internacionalizacion despues es doloroso. Texto hardcodeado, concatenaciones problematicas, formatos de fecha inconsistentes.

**Siempre estructurar para i18n desde el inicio, aunque el V1 sea monolingual.**

### 2. Las imagenes son el 80% del peso

En sitios visuales como turismo, la optimizacion de imagenes tiene el mayor ROI de cualquier mejora de performance.

### 3. WhatsApp > Forms en LATAM

En Latinoamerica, la gente prefiere WhatsApp. Un formulario de contacto tradicional tiene menor conversion que un simple link de WhatsApp.
